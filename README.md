# Паспорт Проекта: «Контроллер роботизированной кисти InMoov»

**Версия:** 2.1.0  
**Статус:** Полное руководство  

## 1. Общее описание

Проект представляет собой комплексную систему для управления пятипальцевой роботизированной кистью *InMoov*. Система реализует четыре независимых режима работы, обеспечивает защиту сервоприводов от механических повреждений и предоставляет пользователю интуитивно понятные интерфейсы взаимодействия (веб-интерфейс, голосовое управление).

Архитектура построена по двухконтроллерной схеме: *ESP32* выступает в роли центрального координатора (*Master*) с веб-сервером, а *Arduino UNO* — в роли исполнительного контроллера (*Slave*), непосредственно управляющего драйвером сервоприводов.

Такая архитектура обеспечивает стабильность: сложная логика и сетевые задачи изолированы на ESP32, а критически важная задача управления серво в реальном времени выполняется на Arduino без задержек.

---

## 2. Ключевые особенности

*   **Четыре режима управления:** Ручной конструктор, БИО-ЭМГ интерфейс, Автономный хват и Голосовое управление.
*   **Двухуровневая система безопасности:** Логика защиты реализована на ESP32 и блокирует анатомически невозможные движения пальцев в режимах "Конструктор" и "Голос".
*   **Интуитивный веб-интерфейс:** Визуальная очередь команд, предустановленные жесты ("Кулак", "Ладонь") и голосовой ввод через браузер.
*   **Модульность:** Система легко расширяется за счет добавления новых режимов или датчиков.

## 4. Подробное описание комплектующих и их назначение

### 4.1. Модуль PCA9685 (16 каналов)

**Назначение:**  
Это 16-канальный ШИМ-драйвер, управляющий сервоприводами. Работает по шине I²C, управляется Arduino Uno. Позволяет эффективно контролировать пять сервоприводов кисти, высвобождая цифровые пины Arduino.

**Характеристики:**  
- Частота ШИМ: до 1600 Гц  
- Ток нагрузки: до 1 А на канал  
- Совместимость с Arduino и Raspberry Pi  

**Примечание:**  
Используйте внешние источники питания (5V/10A) для сервоприводов, чтобы избежать сбоев и перегрева.

---

### 4.2. Пластик PLA Lite (черный, 1 кг)

**Назначение:**  
Материал для 3D-печати каркаса кисти и предплечья. Используется для печати всех механических деталей (фаланг, ладонной пластины, элементов предплечья).

**Характеристики:**  
- Материал: PLA (полилактид)  
- Цвет: матовый чёрный  
- Вес: 1 кг (использованы две катушки)  
- Температура печати: 190–210°C  

**Совет:**  
После печати очистите детали от поддерживающих структур и убедитесь, что внутренние каналы пальцев гладкие и чистые для беспрепятственного прохождения сухожилий.

---

### 4.3. Набор проводов DuPont (60 шт.)

**Назначение:**  
Используются для подключения электронных компонентов (модули, датчики, разъёмы питания). Имеют три типа разъёмов: мама-мама, папа-папа, папа-мама.

**Характеристики:**  
- Длина: 10 см и 20 см  
- Тип изоляции: ПВХ  
- Цветовая маркировка для удобного монтажа  

**Совет:**  
Используйте разноцветные провода для облегчения идентификации соединений (например, синий — GND, красный — VCC, жёлтые — сигналы).

---

### 4.4. Контроллер Arduino Uno R3

**Назначение:**  
Исполнительный контроллер, принимающий команды от ESP32 по I²C и управляющий сервоприводами через PCA9685. Является сердцем исполнительной подсистемы.

**Характеристики:**  
- Процессор: ATmega328P  
- Рабочая частота: 16 МГц  
- Память: 32 КБ Flash, 2 КБ RAM  
- Интерфейсы: I²C, UART, SPI  

**Совет:**  
Установите последнюю версию Arduino IDE и необходимые библиотеки (Adafruit PCA9685 library).

---

### 4.5. Контроллер ESP32-WROOM-32

**Назначение:**  
Центральный координатор системы. Организует Wi-Fi точку доступа, создаёт веб-сервер и осуществляет обработку команд пользователя. Выполняет проверку безопасности (Anti-Jam System).

**Характеристики:**  
- Процессор: Xtensa Dual-core 32-bit CPU  
- Частота: до 240 МГц  
- Память: 4 MB Flash, 520 KB RAM  
- Интерфейсы: Wi-Fi, Bluetooth, I²C, UART  

**Совет:**  
Используйте Arduino IDE с установленным пакетом ESP32 для загрузки прошивки.

## 5. Техническая схема сборки

Сборка делится на механическую и электронную части.

**Шаг 1. Механическая сборка**
1.  Напечатать детали кисти на 3D-принтере и очистить от поддержек.
2.  Соединить фаланги пальцев винтами М2/МЗ, обеспечивая плавный ход.
3.  Собрать каркас предплечья на винтах МЗ, предусмотрев место для установки пяти сервоприводов.
4.  Протянуть сухожилия из плетеного шнура через пальцы: одно для сжатия, другое для разжатия.
5.  Зафиксировать шнуры на качалках сервоприводов, предварительно выставив их в начальное положение (0°).

**Шаг 2. Электронная сборка**
Вся электроника делится на три контура:

1.  **Силовой контур:** Питание (5В/10А) подается на драйвер PCA9685. От него питание идет на VCC сервоприводов. **Важно:** Не питайте сервоприводы напрямую от плат микроконтроллеров!
    *   `PCA9685.VCC` -> Внешний блок питания 5V
    *   `PCA9685.GND` -> Общий GND

2.  **Исполнительный контур (Arduino + PCA9685):**
    *   `PCA9685.SDA` -> `Arduino.A4`
    *   `PCA9685.SCL` -> `Arduino.A5`
    *   `PCA9685.GND` -> `Arduino.GND`
    *   `PCA9685.VCC` (логика) -> `Arduino.5V`

3.  **Коммуникационный контур (ESP32 + Arduino):**
    *   `ESP32.TX` -> `Arduino.RX` (через резистивный делитель напряжения 3.3В -> 5В)
    *   `ESP382.GND` -> `Arduino.GND` (общая земля)
    *   Питание обеих плат (`VIN`) осуществляется параллельно от общей силовой шины 5В.

## 6. Программная реализация

Система состоит из трех независимых прошивок.

**Код 1: Веб-интерфейс (`web/index.html`)**
Предоставляет пользователю графический интерфейс для управления. Позволяет переключать режимы, управлять пальцами вручную, создавать и запускать макросы (последовательности действий), а также использовать голосовые команды.

**Код 2: Координатор (`esp32/main.ino`)**
*   **Функции:** Создает Wi-Fi сеть `InMoov_Hand_AP`, запускает веб-сервер, генерирует HTML-код страницы.
*   **Ключевая роль:** Реализует всю бизнес-логику. При получении команды `/execute` он проверяет её на соответствие правилам системы защиты (*Anti-Jam System*). Если команда валидна, он отправляет её на Arduino по I2C.

**Код 3: Исполнитель (`arduino/servo_controller.ino`)**
*   **Функции:** Инициализирует I2C как Slave-устройство (адрес 0x08), настраивает драйвер PCA9685.
*   **Ключевая роль:** Ожидает команды от ESP32. Получив команду, он переводит угол поворота (0-180°) в значение ШИМ (150-600) и устанавливает соответствующий сигнал на выходе драйвера для нужного сервопривода.

## 7. Режимы работы и логика безопасности

| Параметр | Режим 0: Конструктор | Режим 1: БИО-ЭМГ | Режим 2: Автохват | Режим 3: Голос |
| :--- | :--- | :--- | :--- | :--- |
| **Описание** | Пошаговое создание макросов через веб-интерфейс. | Прямое управление силой мышечных сокращений. | Автоматическое сжатие при обнаружении предмета датчиком HC-SR04. | Управление простыми голосовыми командами. |
| **Интерфейс** | Веб-панель с ползунками и очередью действий. | Подключение ЭМГ-датчиков к аналоговым входам ESP32. | Датчик расстояния, подключенный к Arduino. | Web Speech API браузера. |
| **Логика защиты** | **Полностью активна.** Блокирует запрещенные положения и опасные переходы. | **Отключена.** Обеспечивает полную свободу движений. | **Отсутствует.** Требует полного сжатия/разжатия. | **Полностью активна.** Защищает от неверных команд. |

**Правила защиты (*Anti-Jam System*):**
1.  **Запрещенные статические положения:** Нельзя оставить только средний палец открытым или только средний и большой, когда остальные согнуты.
2.  **Правило сжатой руки:** Из полностью сжатого состояния нельзя разомкнуть только средний палец или только средний и большой.
3.  **Правило разогнутой руки:** В полностью разогнутое состояние нельзя согнуть все пальцы, кроме среднего, или согнуть указательный/безымянный/мизинец, оставляя средний и большой разогнутыми.

## 9. Руководство по запуску

1.  **Прошивка Arduino UNO:** Загрузите файл `arduino/servo_controller.ino`. Выберите плату *Arduino Uno* и нужный COM-порт.
2.  **Прошивка ESP32:** Загрузите файл `esp32/main.ino`. Выберите плату *ESP32 Dev Module* и нужный COM-порт.
3.  **Физическая сборка:** Соберите механическую часть согласно инструкции и подключите электронику по схеме.
4.  **Подключение к системе:**
    *   Найдите Wi-Fi сеть `InMoov_Hand_AP` и подключитесь к ней.
    *   Откройте браузер и перейдите по адресу `http://10.10.10.1`.
    *   Выберите желаемый режим управления.
