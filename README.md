## 🎮 Режимы Управления

### Режим 0️⃣ — Ручной Конструктор (*Constructor Mode*)
**Описание:** Пошаговое управление каждым пальцем через графический интерфейс.
**Особенности:**
*   Управление ползунками для каждого пальца (угол 0-180°).
*   Очередь команд с возможностью редактирования.
*   Добавление паузы между движениями.
*   **Система защиты от повреждений включена по умолчанию.**

### Режим 1️⃣ — БИО-ЭМГ Интерфейс (*Electromyography Interface*)
**Описание:** Интуитивное управление кистью на основе мышечных сокращений пользователя.
**Особенности:**
*   Чтение биоэлектрических сигналов с предплечья.
*   Прямое управление без программных задержек.
*   **Система защиты от повреждений отключена** для обеспечения полной свободы движений.

### Режим 2️⃣ — Автономный Хват (*Autonomous Grasp*)
**Описание:** Автоматическое сжатие кисти при обнаружении предмета датчиком расстояния.
**Особенности:**
*   Датчик расстояния определяет наличие объекта в зоне захвата.
*   Кисть автоматически сжимается до полного закрытия.
*   При удалении предмета кисть разжимается.
*   **Система защиты от повреждений отключена.**

### Режим 3️⃣ — Голосовой Модуль (*Voice Control*)
**Описание:** Управление кистью с помощью голосовых команд через браузер.
**Поддерживаемые команды:**
*   "кулак", "сожми" → сжать все пальцы (180°).
*   "ладонь", "открой" → разогнуть все пальцы (0°).
**Особенности:**
*   Использует Web Speech API браузера.
*   Распознавание русской речи.
*   **Система защиты от повреждений включена.**

## 🛡️ Система Защиты от Повреждений (*Anti-Jam System*)

Программно-электронный комплекс, предотвращающий поломку суставов кисти. Блокирует анатомически невозможные положения пальцев.

### Запрещённые Положения Руки:
Эти комбинации **запрещены** для предотвращения повреждения среднего сустава:
1.  Отогнут только средний палец, остальные согнуты.
2.  Отогнуты только средний и большой пальцы, остальные согнуты.

### Правила Системы:
Правило применяется симметрично как при сгибании, так и при разгибании руки из крайних положений.

| Режим | Конструктор | ЭМГ | Автохват | Голос |
| :--- | :---: | :---: | :---: | :---: |
| **Защита от повреждений** | ✅ ВКЛЮЧЕНА | ❌ ВЫКЛЮЧЕНА | ❌ ВЫКЛЮЧЕНА | ✅ ВКЛЮЧЕНА |
| **Полная свобода** | ❌ НЕТ | ✅ ДА | ✅ ДА | ❌ НЕТ |

## ⚙️ Аппаратное Обеспечение

### Основные компоненты:

| Компонент | Модель | Назначение |
| :--- | :--- | :--- |
| Микроконтроллер | ESP32-WROOM-32 | Wi-Fi сервер, координация, веб-интерфейс |
| Микроконтроллер | Arduino UNO | Исполнитель Slave-контроллер |
| ШИМ Драйвер | PCA9685 | Управление 5 сервоприводами |
| Серво моторы | MG996R (×5) | Приводы пальцев |
| Датчик расстояния | HC-SR04 | Детектирование предметов (Режим 2) |
| Модуль ЭМГ | MyoWare | Считывание биосигналов (Режим 1) |

### Подключение I2C:

**Arduino UNO ↔ PCA9685:**
```
Arduino UNO          PCA9685 Shield
─────────────────────────────────
A4 (SDA) ─────────► SDA
A5 (SCL) ─────────► SCL
GND ──────────────► GND
5V ───────────────► VCC
```

**ESP32 ↔ Arduino UNO:**
```
ESP32                Arduino UNO
────────────────────────────────
GPIO 21 (SDA) ─────► A4 (SDA)
GPIO 22 (SCL) ─────► A5 (SCL)
GND ───────────────► GND
```

## 🚀 Загрузка Кода

### Arduino UNO (`servo_controller.ino`):
1.  Установите библиотеку `Adafruit PWM Servo Driver Library`.
2.  Загрузите скетч `arduino/servo_controller.ino`.

4.  Нажмите *Upload*.

### ESP32 (`main.ino`):
1.  Добавьте в менеджере плат URL: `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`.
2.  Установите плату *ESP32 Dev Module*.
3.  Загрузите скетч `esp32/main.ino`.
4.  Выберите правильный COM-порт и нажмите *Upload*.

## 📊 Протокол I2C Команд

**Адрес Slave устройства на Arduino:** `0x08`

#### Формат команды (2 байта):
```
BYTE 0: Палец (0-5)
├─ 0 = ВСЯ КИСТЬ
├─ 1 = БОЛЬШОЙ ПАЛЕЦ
├─ 2 = УКАЗАТЕЛЬНЫЙ
├─ 3 = СРЕДНИЙ
├─ 4 = БЕЗЫМЯННЫЙ
└─ 5 = МИЗИНЕЦ

BYTE 1: Угол (0-180 градусов)
```

#### Примеры команд:
```cpp
// Сжать указательный палец на 90°
Wire.beginTransmission(0x08);
Wire.write(2);      // Указательный палец
Wire.write(90);     // Угол
Wire.endTransmission();
```

## 🔌 REST API Endpoints (ESP32)

| Endpoint | Метод | Описание |
| :--- | :--- | :--- |
| `/` | GET | Возвращает главную страницу веб-интерфейса |
| `/set_mode` | GET | Переключает режим работы<br>`?m=0..3` |
| `/execute` | GET | Выполняет команду<br>`?f=0..5&a=0..180` |

### Примеры запросов:
```bash
# Переключиться на режим конструктора
curl "http://10.10.10.1/set_mode?m=0"

# Разогнуть мизинец
curl "http://10.10.10.1/execute?f=5&a=0"
```

## 📞 Контакты и Помощь

При возникновении проблем выполните следующие шаги:
1.  Проверьте вывод в **Serial Monitor** обоих микроконтроллеров.
2.  Убедитесь в правильности всех соединений, особенно линий **I2C (SDA/SCL)**.
3.  Перезагрузите оба контроллера.
4.  Проверьте напряжение питания сервоприводов (**5-6В**).
5.  Убедитесь, что адреса устройств совпадают (**Arduino Slave**: `0x08`, **PCA9685**: `0x40`).
